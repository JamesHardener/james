Http  不安全的原因


	Http 不安全的根本原因
	1. 数据没有加密
	2. 无法验证身份
	3. 数据容易被篡改

	密码学的四个目标
	1. 机密性
	2. 完整性
	3. 身份验证
	4. 不可抵赖性
	
	密码学原理公开, 最著名的项目是OpenSSL  包括底层的密码库和命令行工具
	
	随机数
		分类: 
			真正的随机数生成器  硬件生成
			伪随机数生成器		软件生成
			密码学伪随机数		软件生成
			
			算法 随机性 不可预测性 不可重现性
			
			加密单元: 一些基础的密码学算法, 功能单一可靠,通过它们可以构建更多的密码学算法协议 应用程序
			
	基于密码学的Hash算法 的 其它密码学算法
		MAC消息验证码  伪随机数生成器  基于口令的加密算法  数字签名 块密码学加密算法
		
		说明: 基于普通Hash算法实现的应用,比如Hash表 和 校验和 不能用于密码学。
		
		摘要/散列值 /指纹  = hash(消息)
			

	Hash算法的主要特征  5个
	用途  文件比较  身份验证 
	
	安全的密码学Hash算法
		强抗碰撞性  弱抗碰撞性 (强弱相对的,不代表安全程度)  单向性
		
		Hash碰撞: 两个不同的值得到同一摘要值
		
		MD5 不安全的, 违背了强抗碰撞性原则
		
		SHA族类算法: 常用SHA-2
		
		对称加密算法:  分为 块密码算法 和 流密码算法
			一次性密码本: 明文和同样长度的 序列 进行XOR(异或运算) 得到 密文, 密文 和 同样序列进行 XOR 得到原始 明文 
			
			块密码: ECB 模式 存在安全问题
					CBC 模式  
					CTR 模式
		非对称加密算法

	密钥

		TLS/SSL 协议  位于应用层协议 和 TCP协议之间
		密钥协商算法  RSA  DH
		PKI 技术    保证客户端接收到服务端公钥是 真正的服务端密钥
		HTTPS 分为 握手层 和 加密层
			握手: 
				1. 认证
				2. 密码套件协商
				3. 密钥协商
				4. 握手消息完整性校验
			
			加密:
				流密码加密模式  分组加密模式  AEAD模式 (新型加密,一步解决 加密和完整性 )
			
		
		实施HTTPS网站必要条件
			1.证书 和 密钥对
			2.部署 和 配置 HTTPS网站
